# Ankyra 重建總規格

版本日期：2026-03-30
文件用途：本文件作為 Ankyra 專案的單一重建規格書。目標是讓 Codex 或其他工程團隊在「另開新專案」的情況下，只依本文件即可一步步重新建構 Ankyra。

重要前提：
1. 本文件同時描述：
   - 本專案目前已完成的功能與設計
   - 本專案目前所使用的工具與技術
   - 重建時的目標架構
2. 重建目標技術組合明確指定為：
   - 前端：React
   - 後端：Rust
3. 本文件預設正式文件、畫面主要語言與需求文件皆以中文撰寫。


==================================================
一、產品定位
==================================================

產品名稱：Ankyra

一句話定位：
Ankyra 是一個面向知識工作者與管理者的多用戶 AI 工作助理平台，透過 OAuth2 身分登入、對話與記憶持久化、外部信箱與工具整合、工作快照與主動通知，協助使用者整理今日待辦、準備晨會、追蹤工作並保留長期工作脈絡。

主要價值：
1. 將使用者工作脈絡長期保存於自家系統
2. 將對話、記憶、外部同步、通知串成工作閉環
3. 將助理商品化，可申請、升級、定價、治理
4. 先支援 demo fallback，不依賴外部服務也能展示

主要目標客群：
1. 主管
2. 專案負責人
3. 跨部門協調者
4. 高知識密度工作者

第一版主打場景：
1. 郵件整理
2. 今日待辦盤點
3. 晨會摘要
4. 同步後主動通知

1. React
2. TypeScript
3. Vite
4. React Router
5. i18n 由前端自己維護
6. 與後端溝通採 REST API

4.2 後端
1. Rust
2. 建議 Web framework：Axum
3. 建議 async runtime：Tokio
4. 建議序列化：Serde
5. 建議 DB layer：SQLx
6. 建議 PostgreSQL driver：sqlx-postgres
7. 建議 Redis：redis-rs 或 fred
8. 建議 queue/worker：
   - 第一優先：自建 Redis queue + worker loop
   - 或採現成 Rust queue 套件，但要保留 dead-letter / retry / pause-resume / metrics
9. 建議 config：dotenvy + envy / config
10. 建議 auth/session：
   - Cookie + signed session token + DB session

4.3 前後端部署形態
1. React 獨立 dev server / build
2. Rust API server
3. Rust worker process
4. PostgreSQL
5. Redis

4.4 基本原則
1. 前端與後端分離
2. 正式資料保存在自家 DB
3. 外部 provider 只作為授權與資料來源
4. conversation 不等於 memory
5. tool / connector / policy / approval / audit 必須保留

==================================================
十一、Google OAuth 實作要求
==================================================

重建時，Google OAuth 必須實作到可用狀態，而不是只保留 demo。

11.1 provider begin
1. 前端點擊 Google 登入
2. POST /api/auth/oauth/begin
3. 後端回傳 authorization_url
4. 前端 redirect 到 Google

11.2 callback
1. Google redirect 回後端：
   /api/auth/oauth/callback/google
2. 後端交換 token
3. 後端取得 userinfo
4. 建立或綁定 user / oauth_identity / session
5. 後端 redirect 回前端：
   frontend_url?session_token=...

11.3 前端登入狀態
1. 有 session_token：
   - 顯示用戶名稱
   - 按鈕顯示登出
2. 無 session_token：
   - 不顯示用戶名稱
   - 按鈕顯示登入

11.4 建議本機 URI
1. 前端：
   http://127.0.0.1:4174/web
2. 後端：
   http://127.0.0.1:3000
3. Google redirect URI：
   http://127.0.0.1:3000/api/auth/oauth/callback/google
