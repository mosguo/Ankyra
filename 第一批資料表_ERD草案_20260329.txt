Ankyra 第一批資料表 ERD 草案
日期：2026-03-29

一、目的
本文件用於定義 MVP 第一批核心資料表、主要欄位與關聯方向，作為後續 migration、ORM schema 與 API 實作依據。

二、ERD 主線
1. 組織與身分主線
- organizations -> memberships -> users -> oauth_identities

2. 助理主線
- assistant_definitions -> assistant_versions -> assistant_instances

3. 對話主線
- assistant_instances -> conversations -> conversation_events -> conversation_summaries

4. 外部整合主線
- users / organizations -> connector_accounts -> connector_sync_states

5. 產出與通知主線
- jobs -> task_snapshots / briefing_snapshots -> notification_messages

6. 治理主線
- approval_records
- audit_logs
- policies
- tool_policies

六、實作建議
- 先建立 identity、assistant、conversation、connector、snapshot、notification、governance 這七條主線
- migration 順序先從 owner table 開始，再到關聯表
- 表結構先求穩定，不要一開始過度一般化
