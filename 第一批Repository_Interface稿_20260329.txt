Ankyra 第一批 Repository Interface 稿
日期：2026-03-29

一、目的
本文件定義 MVP 第一批 repository 介面草稿，作為資料存取層與 service implementation 的依賴界面。

二、Identity Repositories
1. OrganizationRepository
介面建議：
- findById(organizationId: string)
- findByCode(code: string)
- save(organization)

2. UserRepository
介面建議：
- findById(userId: string)
- findByEmail(email: string)
- save(user)
- updateLastLogin(userId: string, at: Date)

3. MembershipRepository
介面建議：
- findByUserId(userId: string)
- findActiveByUserAndOrganization(userId: string, organizationId: string)
- save(membership)

4. OAuthIdentityRepository
介面建議：
- findByProviderSubject(provider: string, providerSubject: string)
- findByUserId(userId: string)
- save(oauthIdentity)

三、Assistant Repositories

2. AssistantVersionRepository
介面建議：

3. AssistantInstanceRepository
介面建議：

十、實作建議
- repository 介面放在 packages/domain 或 packages/application
- ORM implementation 放在 packages/infrastructure
- repository 返回 domain object 或 persistence model mapping 結果
- 複雜 query 可額外拆 read repository
