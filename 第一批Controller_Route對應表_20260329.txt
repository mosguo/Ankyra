Ankyra 第一批 Controller / Route 對應表
日期：2026-03-29

一、目的
本文件將第一批 API 詳細規格對應到後端 controller 與 route，作為 API 層實作與模組切分依據。

三、建議 Controller 職責
1. AuthController
- 接收 OAuth2 provider 請求
- 驗證 callback 參數
- 呼叫 AuthService
- 回傳 auth context DTO

2. AssistantCatalogController
- 處理 catalog 查詢與篩選條件
- 回傳 assistant catalog DTO

3. AssistantInstanceController
- 建立 instance
- 轉交升級 / 停用命令
- 回傳 instance DTO 與 approval 狀態

4. ConversationController
- 管理 conversation CRUD 最小版
- 轉交訊息給 AIRuntimeService
- 回傳 message result DTO

5. MemoryController
- 僅提供 memory summary 查詢

6. SnapshotController
- 回傳今日 task / briefing snapshot

7. ConnectorController
- 啟動 authorize
- 接 callback
- 查詢與移除連線

8. NotificationController
- 管理通知偏好
- 查詢通知紀錄

9. ApprovalController
- 列出待審批
- 核准或拒絕

10. AuditController
- 提供查詢型 audit API

11. AdminController
- 提供租戶管理首頁摘要與列表

五、建議實作順序
第一波 Controller
- AuthController
- AssistantCatalogController
- AssistantInstanceController
- ConversationController

第二波 Controller
- ConnectorController
- SnapshotController

第三波 Controller

- AuditController

六、實作建議
- Controller 只做驗證、授權、DTO mapping、service 呼叫
- 業務邏輯放 Application Service
- AI orchestration 不放在 controller 內直接寫
