一、實作規劃目標
本文件用於將前五階段的需求與架構，轉換為可開始實作的 MVP 工作計畫。

二、MVP 範圍
1. 帳號與登入
- OAuth2 登入
- Google / Microsoft / WeChat / Twitter provider 架構
- user / organization / membership 基礎建立

2. 助理基礎
- 助理目錄
- 助理申請
- 助理 instance 建立
- 助理版本載入

3. 對話與 AI 核心
- conversation 建立
- conversation event ledger
- OpenAI orchestration 基礎流程
- summary 寫入

4. 外部整合
- Gmail 或 Outlook 二選一
- 郵件整理
- 今日待辦抽取

5. 產出與通知
- task snapshot
- briefing snapshot
- LINE 或 Telegram 二選一通知

6. 管理與治理
- 租戶管理基本頁
- 審批基本流程
- audit log 基本紀錄

四、第一批 API 清單
1. Auth API
- POST /auth/oauth/login
- GET /auth/me
- POST /auth/switch-organization

2. Assistant API
- GET /assistants/catalog
- GET /assistants/catalog/{id}
- POST /assistant-instances
- POST /assistant-instances/{id}/upgrade
- POST /assistant-instances/{id}/suspend

3. Conversation API
- GET /conversations
- POST /conversations
- GET /conversations/{id}
- POST /conversations/{id}/messages

4. Memory / Snapshot API
- GET /memories
- GET /task-snapshots/today
- GET /briefing-snapshots/today

5. Connector API
- GET /connectors
- POST /connectors/{provider}/authorize
- POST /connectors/{provider}/callback
- DELETE /connectors/{id}

6. Notification API
- GET /notification-preferences
- PUT /notification-preferences/{id}
- GET /notification-messages

7. Approval / Audit API
- GET /approvals
- POST /approvals/{id}/approve
- POST /approvals/{id}/reject
- GET /audit-logs
