Ankyra 正式 Demo 導覽順序
日期：2026-03-30

一、開場定位
1. 先開啟首頁，說明 Ankyra 是面向知識工作者與主管的 AI 工作協調平台。
2. 說明目前展示的是 MVP，但已串起登入、助理、對話、同步、待辦、摘要、通知、工作排程與管理觀測。

二、首頁 Dashboard
1. 指出首頁上方的產品定位與目前狀態。
2. 展示 Dashboard 的四個重點：
- 助理目錄
- 今日待辦
- 晨會摘要
- 系統摘要
3. 說明這不是單純聊天機器人，而是主動整理每日工作脈絡的平台。

三、助理目錄
1. 進入「助理目錄」。
2. 展示不同助理類型與等級。
3. 強調助理是商品化能力，不是單一 prompt。
4. 建立一個助理實例，說明未來可依角色與費率配置。

四、我的助理
1. 進入「我的助理」。
2. 展示已開通的助理實例。
3. 說明使用者不是直接使用抽象模型，而是使用自己的助理實例。

五、對話
1. 進入「對話」。
2. 建立一條新 conversation。
3. 展示訊息時間軸。
4. 說明對話的價值不是單次問答，而是持續累積工作脈絡。
5. 點出目前 demo 對話內容已調整成主管晨會前整理情境。

六、今日待辦
1. 進入「待辦」。
2. 展示今日待辦內容。
3. 強調內容偏向主管與專案負責人場景：
- 客戶升級案件
- 專案交付阻塞
- 跨部門採購追蹤

七、晨會摘要
1. 進入「晨會摘要」。
2. 展示已整理好的摘要段落。
3. 說明使用者不必提早進辦公室彙整內容，系統可先產出可報告版本。

八、Connectors
1. 進入「連接器」。
2. 說明這一區在模擬 Gmail / Outlook 等工作來源。
3. 展示：
- 授權模式
- 連線狀態
- 同步狀態
- 下次同步時間
4. 執行一次「立即同步」。
5. 說明同步後會回寫：
- task snapshot
- briefing snapshot
- notification
- job

九、Notifications
1. 進入「通知」。
2. 展示通知偏好與訊息清單。
3. 說明通知不是單純技術事件，而是：
- 每日摘要
- 待辦提醒
- 需要審批

十、Admin
1. 進入「管理端」。
2. 先看管理摘要：
- 待處理審批
- 已完成工作
- 連接器數
- 通知數
3. 再看健康卡：
- health
- readiness
- database
- queue
4. 再看 queue dashboard：
- 佇列狀態
- timeline
- dead-letter
- retry / requeue
5. 打開某一筆 job details。
6. 說明現在不只看到工程 payload，也能看到較業務化的：
- 業務流程
- 連接器帳號
- 排程來源
- 使用者範圍
- 組織範圍

十一、審批與稽核
1. 展示 approval queue。
2. 說明這些不是假設性的管理需求，而是企業導入時的必要能力。
3. 展示 audit logs，說明：
- 連接器同步
- 通知派送
- 審批核准
- 快照刷新
都可被追溯。

十二、收尾訊息
1. 總結 Ankyra 現在已不是概念，而是已具備可展示的 MVP 主線。
2. 強調目前可展示的完整鏈路：
登入 -> 助理 -> 對話 -> 連接器 -> 同步 -> 待辦/摘要 -> 通知 -> Job/Schedule -> Queue/Admin
3. 說明下一步可再往：
- 真實 OAuth
- 真實 Gmail / Outlook
- 正式 DB / Redis / Worker 環境
- 商用級治理與權限
繼續推進。
